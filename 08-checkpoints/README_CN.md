# 检查点与回退

检查点允许您保存会话状态并在 Claude Code 会话中回退到之前的点。这对于探索不同的方法、从错误中恢复或比较替代方案非常有价值。

## 概述

检查点允许您保存会话状态并回退到之前的点，从而实现安全的实验和多方法探索。它们是会话状态的快照，包括：
- 所有交换的消息
- 进行的文件修改
- 工具使用历史
- 会话上下文

检查点在探索不同方法、从错误中恢复或比较替代方案时非常有价值。

## 关键概念

| 概念 | 描述 |
|---------|-------------|
| **检查点** | 包括消息、文件和上下文的会话状态快照 |
| **回退** | 返回到之前的检查点，丢弃后续更改 |
| **分支点** | 从中探索多种方法的检查点 |

## 访问检查点

您可以通过两种主要方式访问和管理检查点：

### 使用键盘快捷键
按两次 `Esc`（`Esc` + `Esc`）打开检查点界面并浏览已保存的检查点。

### 使用斜杠命令
使用 `/rewind` 命令（别名：`/checkpoint`）快速访问：

```bash
# 打开回退界面
/rewind

# 或使用别名
/checkpoint
```

## 回退选项

当您回退时，会看到一个包含五个选项的菜单：

1. **恢复代码和会话** -- 将文件和消息都恢复到该检查点
2. **仅恢复会话** -- 仅回退消息，保持当前代码不变
3. **仅恢复代码** -- 仅恢复文件更改，保持完整的会话历史
4. **从此处总结** -- 将从此点开始的会话压缩成 AI 生成的摘要，而不是丢弃它。原始消息保留在转录中。您可以选择提供指令来聚焦特定主题。
5. **算了** -- 取消并返回当前状态

## 自动检查点

Claude Code 自动为您创建检查点：

- **每次用户提示** - 每次用户输入时创建新检查点
- **持久化** - 检查点跨会话保留
- **自动清理** - 检查点在 30 天后自动清理

这意味着您始终可以回退到会话中的任何前一点，从几分钟前到几天前。

## 使用场景

| 场景 | 工作流 |
|----------|----------|
| **探索方法** | 保存 → 尝试 A → 保存 → 回退 → 尝试 B → 比较 |
| **安全重构** | 保存 → 重构 → 测试 → 如果失败：回退 |
| **A/B 测试** | 保存 → 设计 A → 保存 → 回退 → 设计 B → 比较 |
| **错误恢复** | 发现问题 → 回退到最后良好状态 |

## 使用检查点

### 查看和回退

按两次 `Esc` 或使用 `/rewind` 打开检查点浏览器。您将看到所有可用检查点的列表及其时间戳。选择任何检查点以回退到该状态。

### 检查点详情

每个检查点显示：
- 创建时的时间戳
- 修改的文件
- 会话中的消息数量
- 使用的工具

## 实际示例

### 示例 1：探索不同方法

```
用户：让我们为 API 添加缓存层

Claude：我将为您的 API 端点添加 Redis 缓存...
[在检查点 A 进行更改]

用户：实际上，让我们尝试内存缓存代替

Claude：我将回退以探索不同的方法...
[用户按 Esc+Esc 并回退到检查点 A]
[在检查点 B 实现内存缓存]

用户：现在我可以比较两种方法了
```

### 示例 2：从错误中恢复

```
用户：将身份验证模块重构为使用 JWT

Claude：我将重构身份验证模块...
[进行大量更改]

用户：等等，那破坏了 OAuth 集成。让我们回去。

Claude：我将帮助您回退到重构之前...
[用户按 Esc+Esc 并选择重构前的检查点]

用户：这次让我们尝试更保守的方法
```

### 示例 3：安全实验

```
用户：让我们尝试以函数式风格重写这段代码
[在实验前创建检查点]

Claude：[进行实验性更改]

用户：测试失败了。让我们回退。
[用户按 Esc+Esc 并回退到检查点]

Claude：我已经回退了更改。让我们尝试不同的方法。
```

### 示例 4：分支方法

```
用户：我想比较两种数据库设计
[记录检查点 - 称之为"开始"]

Claude：我将创建第一个设计...
[实现 Schema A]

用户：现在让我回去尝试第二种方法
[用户按 Esc+Esc 并回退到"开始"]

Claude：现在我将实现 Schema B...
[实现 Schema B]

用户：太好了！现在我有了两种 schema 可供选择
```

## 检查点保留

Claude Code 自动管理您的检查点：

- 检查点在每次用户提示时自动创建
- 旧检查点保留最多 30 天
- 检查点自动清理以防止无限存储增长

## 工作流模式

### 用于探索的分支策略

探索多种方法时：

```
1. 从初始实现开始 → 检查点 A
2. 尝试方法 1 → 检查点 B
3. 回退到检查点 A
4. 尝试方法 2 → 检查点 C
5. 比较 B 和 C 的结果
6. 选择最佳方法并继续
```

### 安全重构模式

进行重大更改时：

```
1. 当前状态 → 检查点（自动）
2. 开始重构
3. 运行测试
4. 如果测试通过 → 继续工作
5. 如果测试失败 → 回退并尝试不同方法
```

## 最佳实践

由于检查点是自动创建的，您可以专注于工作而无需担心手动保存状态。但请记住这些做法：

### 有效使用检查点

✅ **应该：**
- 在回退之前查看可用检查点
- 当想要探索不同方向时使用回退
- 保留检查点以比较不同方法
- 了解每个回退选项的作用（恢复代码和会话、仅恢复会话、仅恢复代码或总结）

❌ **不应该：**
- 仅依赖检查点进行代码保存
- 期望检查点跟踪外部文件系统更改
- 使用检查点替代 git 提交

## 配置

您可以在设置中切换自动检查点：

```json
{
  "autoCheckpoint": true
}
```

- `autoCheckpoint`：启用或禁用每次用户提示时自动创建检查点（默认：`true`）

## 限制

检查点有以下限制：

- **不跟踪 Bash 命令更改** - 如 `rm`、`mv`、`cp` 等文件系统操作不会在检查点中捕获
- **不跟踪外部更改** - 在 Claude Code 之外进行的更改（在编辑器、终端等中）不会被捕获
- **不能替代版本控制** - 使用 git 进行代码库的永久、可审计的更改

## 故障排除

### 检查点丢失

**问题**：未找到预期的检查点

**解决方案**：
- 检查检查点是否被清除
- 验证设置中启用了 `autoCheckpoint`
- 检查磁盘空间

### 回退失败

**问题**：无法回退到检查点

**解决方案**：
- 确保没有未提交的更改冲突
- 检查检查点是否损坏
- 尝试回退到不同的检查点

## 与 Git 集成

检查点补充（但不能替代）git：

| 功能 | Git | 检查点 |
|---------|-----|-------------|
| 范围 | 文件系统 | 会话 + 文件 |
| 持久性 | 永久 | 基于会话 |
| 粒度 | 提交 | 任意点 |
| 速度 | 较慢 | 即时 |
| 共享 | 是 | 有限 |

一起使用：
1. 使用检查点进行快速实验
2. 使用 git 提交进行最终更改
3. 在 git 操作前创建检查点
4. 将成功的检查点状态提交到 git

## 快速入门指南

### 基本工作流

1. **正常工作** - Claude Code 自动创建检查点
2. **想要回去？** - 按两次 `Esc` 或使用 `/rewind`
3. **选择检查点** - 从列表中选择以回退
4. **选择要恢复的内容** - 选择恢复代码和会话、仅恢复会话、仅恢复代码、从此处总结或取消
5. **继续工作** - 您回到了那个点

### 键盘快捷键

- **`Esc` + `Esc`** - 打开检查点浏览器
- **`/rewind`** - 访问检查点的替代方式
- **`/checkpoint`** - `/rewind` 的别名

## 何时回退：上下文监控

检查点让您可以回去 — 但您怎么知道*何时*应该回去？随着会话增长，Claude 的上下文窗口被填满，模型质量悄然下降。您可能正在从半盲模型中交付代码而不自知。

**[cc-context-stats](https://github.com/luongnv89/cc-context-stats)** 通过向您的 Claude Code 状态栏添加实时**上下文区域**来解决这个问题。它跟踪您在上下文窗口中的位置 — 从 **Plan**（绿色，安全地规划和编码）到 **Code**（黄色，避免开始新计划）再到 **Dump**（橙色，收尾并回退）。当您看到区域转换时，您就知道是时候检查点并重新开始，而不是在输出质量下降时继续推进。

## 总结

检查点是 Claude Code 中的自动功能，让您可以安全地探索不同方法而不必担心丢失工作。每次用户提示都会自动创建新检查点，因此您可以回退到会话中的任何前一点。

主要优势：
- 无畏地实验多种方法
- 快速从错误中恢复
- 并排比较不同的解决方案
- 安全地与版本控制系统集成

记住：检查点不能替代 git。使用检查点进行快速实验，使用 git 进行永久代码更改。
